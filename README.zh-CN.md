# 口播大纲动效剪辑 Skill

这个 skill 用来帮助 Codex 在 ChatCut 里剪辑中文口播视频。

它主要解决四件事：

1. 按口播内容添加大纲节点、标题节点、关键词弹出和 MG 动画。
2. 节点必须跟说话时间对齐，说到哪个词，哪个词再弹出。
3. 先检查主视频比例和裁切，避免竖屏口播被拉伸、挤压或套用错误横屏裁切。
4. 如果视频没有字幕，先补字幕，再安排 MG 位置，避免挡字幕。

## 为什么 SKILL.md 用英文

`SKILL.md` 主要是给 Codex 执行时读取的。英文指令通常更容易保持稳定、精确，尤其是涉及工具调用、时间线、字幕、Motion Graphics 这些流程时。

但你不需要用英文理解它，所以这个仓库保留中文说明文件，方便你检查这个 skill 到底会做什么。

## 使用场景

你可以这样说：

```text
帮我分析这个口播视频，按说话内容添加大纲/标题节点和MG动画；如果没有字幕，先补字幕。
```

## 固定参考图

这个 skill 已经把你确认过的两张参考图存入 `assets/`，下次做标题节点和罗列节点时，不应该只凭“图二风格 / 图三风格”这句话复述，而要实际打开参考图检查。

- `assets/title-node-reference.png`：标题 / 章节节点风格。粉色粗标题，白色描边，黑色阴影，位置偏左上，下方可加简短支撑行，比如 `✅ 接情绪` / `❌ 直接接内容`。
- `assets/list-term-reference.png`：罗列观点 / 名词 / 能力词风格。黄色粗字，黑色描边，无卡片背景，两列排布，放在画面下半区但避开字幕。

## 封面规则

封面图生成功能已经从这个 skill 中移除。

默认不要在剪辑开头插入封面图、标题卡或开头静帧。如果用户只是要封面图，应该作为单独图片任务处理，不混进视频剪辑流程。只有用户明确要求“把片头卡插入视频”时，才按视频片头处理，并重新同步后面的字幕和 MG 时间。

## 默认剪辑原则

- 节点跟口播内容对齐，不按固定时间一次性弹出。
- 罗列多个概念时，每个概念分别查 transcript 时间，逐个出现。
- 方法论节点要保留到最后一个子点说完再消失。
- 有对比时，正向内容用 `✅`，反向内容用 `❌`。
- MG 避开人脸、嘴部、重要手势和字幕。
- 标题节点必须参考 `assets/title-node-reference.png`，罗列 / 名词节点必须参考 `assets/list-term-reference.png`。
- 竖屏 9:16 主视频必须先检查几何参数，不能被横屏导入参数拉伸或裁切变形。
- 默认输出是可预览的 ChatCut 项目，不主动导出 MP4。

## 维护规则

以后更新这个 skill 时，必须同步更新 `SKILL.md`、`README.md`、`README.zh-CN.md` 和 `ITERATIONS.md`。如果视觉参考图变化，也要同步更新 `assets/` 和中英文说明，不能只改执行指令。
